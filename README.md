# 🔮 RAG Document Q&A Chatbot

A local Retrieval-Augmented Generation (RAG) chatbot that lets you upload your own documents and ask questions about them in natural language. Answers are grounded in your files and cite the exact source they came from.

Built with **Google Gemini**, **LangChain**, **FAISS**, and **Streamlit**.

---

## ✨ Features

- **Multi-format ingestion** — upload PDF, TXT, DOCX, and Markdown files
- **Semantic search** — chunks are embedded with `sentence-transformers/all-MiniLM-L6-v2` and indexed in FAISS for fast similarity retrieval
- **Gemini LLM** — answers are generated by Google Gemini 3.5 Flash
- **Source citations** — every answer shows which file (and page) it came from
- **Sliding-window memory** — the last N conversation turns are kept in context so follow-up questions work naturally
- **Static two-panel UI** — permanent left panel for document management, right panel for chat; no collapsing sidebars
- **Tunable settings** — temperature, top-K retrieval, memory window, and chunk size are all adjustable at runtime

## 🏗 Architecture

```
User uploads files
       │
       ▼
┌─────────────────────┐
│  DocumentProcessor  │  load → chunk → embed → FAISS index
└────────┬────────────┘
         │ retriever
         ▼
┌─────────────────────┐     ┌───────────────────┐
│     RAGEngine       │────▶│  SessionMemory    │  sliding window history
│  (LangChain chain)  │     └───────────────────┘
└────────┬────────────┘
         │ prompt = history + context + question
         ▼
   Google Gemini 3.5 Flash
         │
         ▼
   Answer + source citations
```

**Three core classes in `rag.py`:**

| Class | Responsibility |
|---|---|
| `DocumentProcessor` | Load files, split into chunks, embed with HuggingFace, build FAISS index |
| `SessionMemory` | Store conversation history, expose a sliding window of the last N turns |
| `RAGEngine` | Retrieve relevant chunks, assemble prompt, call Gemini, persist to memory |

## 🚀 Getting Started

### Prerequisites

- Python 3.11 or higher
- [uv](https://docs.astral.sh/uv/) — fast Python package manager
- A [Google AI Studio](https://aistudio.google.com/) API key (free tier works)

### 1. Clone the repository

```bash
git clone https://github.com/harshilkareliya/QA_Chatbot.git
cd QA_Chatbot
```

### 2. Install uv (if you don't have it)

```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 3. Create the virtual environment and install dependencies

```bash
uv sync
```

This reads `pyproject.toml` and `uv.lock` and installs everything into `.venv` automatically.

### 4. Set your API key

Create a `.env` file in the project root:

```env
GEMINI_API_KEY=your_google_ai_studio_key_here
```

> Get your free key at [https://aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)

### 5. Run the app

```bash
uv run streamlit run streamlit/app.py
```

Then open [http://localhost:8501](http://localhost:8501) in your browser.

## 🖥 Usage

1. **Upload documents** — use the left panel to upload one or more PDF, TXT, DOCX, or MD files
2. **Adjust settings** (optional) — expand the ⚙️ Settings panel to tune temperature, retrieval depth, memory window, and chunk size
3. **Click ⚡ Index** — the app chunks your files, embeds them, and builds a FAISS vector index
4. **Ask questions** — type any question in the chat input; answers will cite which file they came from
5. **New Chat** — click 💬 New Chat to reset the conversation history without losing the index
6. **Clear All** — removes all uploaded files and resets everything

## ⚙️ Configuration

### Runtime settings (in the UI)

| Setting | Default | Description |
|---|---|---|
| Temperature | 0.3 | LLM creativity — lower is more factual |
| Top-K Retrieval | 4 | Number of chunks retrieved per query |
| Memory Window | 5 | Past conversation turns kept in context |
| Chunk Size | 1000 | Characters per document chunk |

### Streamlit config (`.streamlit/config.toml`)

```toml
[server]
fileWatcherType = "none"   # Suppresses torchvision warning spam in terminal

[browser]
gatherUsageStats = false
```

## 🔧 Troubleshooting

**`GEMINI_API_KEY not set` error**
Make sure your `.env` file is in the project root (same level as `pyproject.toml`), not inside the `streamlit/` folder.

**`ModuleNotFoundError: No module named 'torchvision'` in terminal**
This is harmless — Streamlit's file watcher scans all of `transformers`' submodules, some of which require `torchvision`. The `.streamlit/config.toml` setting `fileWatcherType = "none"` silences it. Your app functions correctly regardless.

**Slow first response after indexing**
The HuggingFace embedding model (`all-MiniLM-L6-v2`) is downloaded on first use (~90 MB) and cached locally. Subsequent runs are fast.

**Out of memory with large documents**
Reduce the Chunk Size slider to 500 and re-index. Alternatively, increase chunk overlap to 0 for less redundancy.
